# 概述

双指针作为比赛常用技巧，常常以 $O(n)$ 的复杂度处理一众 Ad-Hoc 问题，本文试图以最简单易懂的语言，描述其使用策略。

## 流派与分类

双指针分为：
### 1. 对撞指针（Collision）

当涉及（1）数组有序（或存在单调性）或者（2）涉及两数组合，即 $a_i+a_j$ 等操作时，令左指针位于数组下标首位，右指针位于数组下标末位，对撞缩小范围以统计答案。

```cpp
while(l < r){
	if(Condition A) ..., l++;
	else if(Condition B) ..., r--;
} //在循环中记得统计答案
```

### 2.快慢指针（Sliding Window）

当涉及“满足条件的连续子数组（最长/最短/计数）”时，考虑快慢指针，类比毛毛虫。

```cpp
int l = 1;
for(int r = 1; r <= n; ++r){
	add(a[r]);  // 加入a[r]到窗口
	while(!check(Condition A)) remove(a[r]), l++; //不满足剔除
}
```


## 坑点

在两种双指针操作中，都存在一些易错坑点：
	① 移动 $l, r$ 必须产生确定性的结果，即数组需要具备一定的单调性。
	② 滑动窗口（对撞指针）的 `check()` 条件最好加上 `l <= r`。
	③ 注意答案统计的时机，需要根据实际情况进行讨论。
